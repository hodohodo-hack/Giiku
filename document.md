# Gi育 (Giiku) プロジェクト仕様書 & 開発ガイド

## 1. プロジェクト概要
「Gi育（ぎいく）」は、日々の開発（Git操作）をキャラクターの育成体験に変換するCLIツールです。
特定のレポジトリに依存せず、ローカルマシン全体でのGitアクションをフックして、グローバルにキャラクターを成長させます。

### 開発の目的
- 開発モチベーションの維持・向上
- Gitコマンド操作の習慣化
- ターミナル環境への愛着形成

---

## 2. システムアーキテクチャ
プロジェクトは、メンテナンス性と拡張性を高めるため、**依存性注入 (Dependency Injection)** と **モジュール化** を重視した設計になっています。

### 主要ディレクトリ構成
- `src/core/`: ビジネスロジック（育成エンジン、称号判定、セットアップ管理）
- `src/infra/`: 外部依存の実装（Git情報取得、データ保存、描画エンジン）
- `src/assets/`: 静的データ定義
  - `titles/`: 各称号の定義ファイル群
  - `skins/`: 各スキンの定義ファイル群
  - `translations.ts`: 多言語対応（日/英）
  - `config.ts`: ゲームバランス調整用の閾値設定
- `src/components/`: React (Ink) によるUIコンポーネント

---

## 3. 育成システム仕様

### コンディション管理
前回の操作からの経過時間に応じてパラメータが減衰し、特定のGitコマンドで回復します。
- **満腹度 (Satiety)**: 1時間ごとに -2%。`git commit` で +20%。
- **ツヤ (Luster)**: 1時間ごとに -1%。`git push` で +15%。

### 視覚的演出
- **オーラ (Aura)**: ツヤが 40% 以上で発生。80% 以上でより豪華な粒子に変化。
- **コンディション変化**: 満腹度が 30% 以下になると、挨拶や見た目が変化。

### 称号システム
ユーザーの行動履歴から自動判定されます。
- **実績称号**: `早起きのエンジニア🌅`（朝コミット）、`デバッグの鬼👹`（fixコミット多）など。
- **動的称号**: `現在：慎重派🐢`（diff多用）、`現在：炎上中🔥`（深夜の連続コミット）など。

---

## 4. アンロック（収集）要素
特定の条件を満たすことで、新しいスキン（キャラクター）が解放されます。
- `SLIME-GENUS`: 初期解放
- `MECHA-GENUS`: 累計10コミット
- `GHOST-GENUS`: 累計50コミット
- `PLANT-GENUS`: 活動3日以上

---

## 5. UI/UX ユーザー体験

### 4つのタッチポイント
1.  **起動時**: ターミナルを開いた瞬間にキャラクターが挨拶 (`--startup`)
2.  **日常**: `git status` 等の後に1行でステータスを表示 (`--status-line`)
3.  **成功時**: `git commit` 直後にリッチな演出で祝福 (`--commit-reaction`)
4.  **管理**: `giiku` コマンドでフル画面TUIモードを起動。検索や着せ替えが可能。

### TUIモードの操作
- **左右キー**: スキンの切り替え（獲得済みのみ）
- **`/` キー**: スキンの曖昧検索モード
  - **上下キー**: 検索候補の選択（スクロール対応リスト）
  - **Enter**: 選択を確定
  - **Esc**: キャンセル
- **`q` キー**: 終了

## 6. 開発者向けガイド

### 新しい称号を追加する
1. `src/assets/titles/` に新ファイル（例: `master.ts`）を作成。
2. `TitleDefinition` インターフェースを実装。
3. `src/assets/titles/index.ts` の `TITLE_DEFINITIONS` 配列に追加。

### 新しいスキンを追加する
1. `src/assets/skins/` に新ファイル（例: `dragon.ts`）を作成。
2. アスキーアートと解放条件 (`checkUnlock`) を定義。
3. `src/assets/skins/definitions.ts` の `SKIN_DEFINITIONS` 配列に追加。

### デバッグ用コマンド
開発を効率化するための隠しフラグが用意されています。
- `giiku --help --debug`: デバッグ用ヘルプの表示
- `giiku --god-mode`: 全スキンを一時的に解放してTUI起動
- `giiku --debug-unlock`: アンロック演出のシミュレーション
- `giiku --hook starve`: 強制的にお腹を空かせる
- `giiku --hook rust`: 強制的にツヤを失わせる

---

## 7. セットアップ
プロジェクトをクローン後、以下を実行して開発環境を構築します。
```bash
npm install
npm run build
npm run link-cli
giiku --setup --ja  # 日本語モードでインストール
```
